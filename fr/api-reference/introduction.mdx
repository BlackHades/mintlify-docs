## Vue d'ensemble

Payfonte fournit des API pour trois capacités principales :

- **Collectes** : créer des checkouts de paiement, lancer des direct charges et vérifier les transactions.
- **Décaissements** : valider des bénéficiaires, demander des virements, lister les décaissements et vérifier leur statut.
- **Portefeuilles** : récupérer les soldes de portefeuille par `country` et `currency`.

La spécification API définit **11 opérations** sur les flux de collecte, de décaissement et de portefeuille.

## URL de base

Utilisez le bon serveur selon votre environnement :

| Environnement | URL de base |
| --- | --- |
| Sandbox | `https://sandbox-api.payfonte.com` |
| Production | `https://api.payfonte.com` |

## Authentification

Tous les endpoints exigent deux en-têtes de clé API, et non une authentification Bearer :

- `client-id`
- `client-secret`

```bash
curl --request GET 'https://sandbox-api.payfonte.com/payments/v1/checkouts' \
  --header 'client-id: <your-client-id>' \
  --header 'client-secret: <your-client-secret>'
```

## Cartographie des endpoints

### Collectes

| Méthode | Chemin | Objet |
| --- | --- | --- |
| `POST` | `/payments/v1/checkouts` | Générer une URL de checkout |
| `GET` | `/payments/v1/checkouts` | Lister les transactions de checkout (paginées) |
| `POST` | `/payments/v1/payments/direct-charge` | Lancer un direct charge via un provider |
| `GET` | `/payments/v1/payments/direct-charge/{provider}/properties` | Récupérer les propriétés spécifiques au provider |
| `GET` | `/payments/v1/payments/verify/{reference}` | Vérifier une transaction de collecte |

### Décaissements

| Méthode | Chemin | Objet |
| --- | --- | --- |
| `GET` | `/billing/v1/transfer-recipients/{provider}/properties` | Récupérer les propriétés du bénéficiaire/provider |
| `POST` | `/billing/v1/transfer-recipients/validate` | Valider un bénéficiaire de virement |
| `POST` | `/billing/v1/disbursements` | Demander un décaissement |
| `GET` | `/billing/v1/disbursements` | Lister les transactions de décaissement (paginées) |
| `GET` | `/billing/v1/disbursements/verify/{reference}` | Vérifier le statut d'un décaissement |

### Portefeuilles

| Méthode | Chemin | Objet |
| --- | --- | --- |
| `GET` | `/billing/v1/wallets` | Récupérer les soldes des portefeuilles |

## Conventions communes des requêtes

- **Format des montants** : envoyez `amount` dans la plus petite dénomination (sous-unité).
  Exemple : `100.00` USD s'envoie sous la forme `10000`.
- **Devise** : code sur 3 caractères (par exemple `NGN`, `USD`, `XOF`, `XAF`).
- **Pays** : code sur 2 caractères (par exemple `NG`, `US`, `CI`, `SN`).
- **Référence** : la référence générée par le marchand doit être unique par requête.
- **Surcharge webhook** : certains endpoints acceptent une URL de webhook spécifique à la requête.

## Flux habituels

### Flux de collecte

1. Créez un checkout (`POST /payments/v1/checkouts`) ou lancez un direct charge (`POST /payments/v1/payments/direct-charge`).
2. Recevez les mises à jour via webhook et/ou interrogez la vérification (`GET /payments/v1/payments/verify/{reference}`).
3. Utilisez l'endpoint de liste (`GET /payments/v1/checkouts`) pour le rapprochement et le reporting.

### Flux de décaissement

1. Récupérez les propriétés du provider si nécessaire puis validez le bénéficiaire (`POST /billing/v1/transfer-recipients/validate`).
2. Demandez le décaissement (`POST /billing/v1/disbursements`).
3. Vérifiez le statut (`GET /billing/v1/disbursements/verify/{reference}`) et rapprochez avec l'endpoint de liste (`GET /billing/v1/disbursements`).

## Pagination et filtres

Les endpoints de liste prennent en charge la pagination avec `page` et `limit`.
Les endpoints de liste de collecte et de décaissement prennent aussi en charge les filtres de date (`dateFrom`, `dateTo`).
