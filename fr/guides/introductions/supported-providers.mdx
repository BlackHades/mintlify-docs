Utilisez cette page pour choisir le bon slug provider selon le pays, le type de transaction et la devise de votre intégration.

<Info>
  Cette page est la source de vérité pour la couverture provider, les slugs, les devises prises en charge, les exigences fetch-properties et les limites min/max publiées.
</Info>

## En un coup d'œil

<CardGroup cols={2}>
  <Card title="14 marchés" icon="globe-africa">
    La couverture provider est actuellement publiée pour 14 pays d'Afrique de l'Ouest, de l'Est, centrale et australe.
  </Card>
  <Card title="43 slugs provider" icon="fingerprint">
    Envoyez la valeur exacte du `Slug` dans vos payloads API lorsque vous renseignez le champ `provider`.
  </Card>
  <Card title="37 slugs double flux" icon="right-left">
    La plupart des slugs prennent en charge à la fois la `Collecte` et le `Décaissement`, avec quelques exceptions selon le marché.
  </Card>
  <Card title="1 fetch requis" icon="sliders">
    `bank-transfer-nigeria` est actuellement le seul provider listé qui requiert un appel fetch-properties.
  </Card>
</CardGroup>

<Warning>
  Payfonte ne prend pas en charge les montants décimaux. Envoyez uniquement des montants entiers en sous-unités. Par exemple, `100.50 NGN` devient `10050`. Voir [Spécification des montants](/fr/guides/introductions/amount-specification).
</Warning>

<Note>
  Un `—` dans les colonnes min/max signifie qu'aucune limite n'est actuellement publiée sur cette page. Les limites réelles sont généralement fixées par le provider ou le MNO et peuvent varier selon le niveau de compte de l'utilisateur final ; ne considérez donc pas `—` comme une garantie d'absence de restriction.
</Note>

## Guide des colonnes

<AccordionGroup>
  <Accordion title="Provider" icon="building" defaultOpen>
    Nom lisible du provider, comme MTN MoMo, Orange, Wave, Bank Transfer ou M-Pesa.
  </Accordion>
  <Accordion title="Slug" icon="tag">
    Identifiant provider machine-readable utilisé dans les appels API, par exemple `mtn-momo-ghana`.
  </Accordion>
  <Accordion title="Devise" icon="coins">
    Devise de transaction attendue pour le pays et la ligne provider concernés.
  </Accordion>
  <Accordion title="Type" icon="right-left">
    - `Collecte` : flux de collecte ou de débit
    - `Décaissement` : flux de payout ou de transfert
  </Accordion>
  <Accordion title="Fetch Properties" icon="sliders">
    `Yes` signifie qu'il peut être nécessaire d'appeler un endpoint provider-properties avant d'initier la requête.
  </Accordion>
  <Accordion title="Min. Amount / Max. Amount" icon="hashtag">
    Seuils de transaction publiés lorsqu'ils sont définis. Les valeurs suivent les règles de sous-unités décrites dans [Spécification des montants](/fr/guides/introductions/amount-specification).
  </Accordion>
</AccordionGroup>

<Info>
  La disponibilité des providers peut évoluer selon le marché, la conformité ou la disponibilité des partenaires. Confirmez toujours votre configuration actuelle dans le dashboard avant le go-live.
</Info>
