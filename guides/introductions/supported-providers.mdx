<Info>
  This page is the source of truth for provider coverage, provider slugs, and min/max transaction limits.
</Info>

Use this page to pick the right provider slug for your integration by country and transaction type.

## How To Use This Page

<CardGroup cols={3}>
  <Card title="Match Country + Type" icon="location-dot">
    Select a country tab and confirm whether the provider supports `collection`, `disbursement`, or both.
  </Card>
  <Card title="Use Exact Slug" icon="fingerprint">
    The `Slug` value is what your API payloads should send in the `provider` field.
  </Card>
  <Card title="Check Limits Early" icon="scale-balanced">
    Review min/max amount ranges where available to avoid failed requests.
  </Card>
</CardGroup>

## Column Guide

<AccordionGroup>
  <Accordion title="Provider" icon="building" defaultOpen>
    Human-readable provider name (for example, MTN MoMo, Orange, Wave, Bank Transfer).
  </Accordion>
  <Accordion title="Slug" icon="tag">
    Machine-readable provider identifier used in API calls (for example, `mtn-momo-ghana`).
  </Accordion>
  <Accordion title="Type" icon="right-left">
    - `collection`: collection/charge flow
    - `disbursement`: disbursement flow
  </Accordion>
  <Accordion title="Fetch Properties" icon="sliders">
    `Yes` means you may need to call a provider-properties endpoint before initiating a request.
  </Accordion>
  <Accordion title="Min. Amount / Max. Amount" icon="hashtag">
    Transaction thresholds where currently defined. Amount values follow the currency minor-unit rules in [Amount Specification](/guides/introductions/amount-specification).
  </Accordion>
</AccordionGroup>

## Quick API Examples

<Warning>
  Payfonte does not support decimal amounts. Send integer amounts only in minor units (for example, `100.50 NGN` becomes `10050`). See [Amount Specification](/guides/introductions/amount-specification).
</Warning>

<CodeGroup>
```json Collection example (provider slug)
{
  "provider": "mtn-momo-ghana",
  "amount": 10000,
  "currency": "GHS",
  "country": "GH",
  "reference": "ORDER-1001",
  "customerInput": {
    "phoneNumber": "233xxxxxxxxx"
  }
}
```

```json Disbursement example (provider slug)
{
  "provider": "bank-transfer-nigeria",
  "country": "NG",
  "currency": "NGN",
  "account": {
    "accountNumber": "0123456789",
    "bankCode": "044"
  }
}
```
</CodeGroup>

<Info>
  Provider availability can change by market, compliance policy, or partner uptime. Confirm current setup in your dashboard before going live.
</Info>
