# Disbursements

Payfonte enables you to send disbursements to mobile money wallets and bank accounts across 14+ African markets through a single **payment orchestration API**. Leverage our **mobile money API** integrations including **MTN MoMo API**, **M-Pesa API**, **Airtel Money API**, and **Wave Money integration** for seamless disbursements.

## Quick Start

### 1. Create a Disbursement

```bash
curl --location 'https://sandbox-api.payfonte.com/payments/v1/disbursements' \
  --header 'client-id: YOUR_CLIENT_ID' \
  --header 'client-secret: YOUR_CLIENT_SECRET' \
  --header 'Content-Type: application/json' \
  --data '{
    "reference": "disbursement-001",
    "amount": 10000,
    "currency": "NGN",
    "country": "NG",
    "provider": "bank-transfer-nigeria",
    "recipient": {
      "name": "John Doe",
      "phoneNumber": "08012345678",
      "bankCode": "058",
      "accountNumber": "0123456789"
    },
    "webhook": "https://yoursite.com/webhooks/payfonte"
  }'
```

### 2. Response

```json
{
  "success": true,
  "data": {
    "disbursementId": "dis_abc123xyz",
    "reference": "disbursement-001",
    "amount": 10000,
    "currency": "NGN",
    "status": "pending",
    "provider": "bank-transfer-nigeria"
  }
}
```

### 3. Handle Webhook

```json
{
  "event": "disbursement.completed",
  "data": {
    "reference": "disbursement-001",
    "disbursementId": "dis_abc123xyz",
    "amount": 10000,
    "currency": "NGN",
    "status": "successful",
    "completedAt": "2025-02-06T10:30:00Z"
  }
}
```

## Supported disbursement Methods

### Mobile Money

| Country | Providers | Currency |
|---------|-----------|----------|
| Nigeria | MTN MoMo, Airtel SmartCash, Opay Wallet | NGN |
| Ghana | MTN MoMo, AirtelTigo, Telecel | GHS |
| Kenya | M-Pesa | KES |
| Tanzania | M-Pesa, Airtel, Halopesa, Tigo | TZS |
| Ivory Coast | Orange, MTN, Wave, Moov | XOF |
| Senegal | Orange, Wave | XOF |
| Cameroon | Orange, MTN | XAF |
| Burkina Faso | Orange, Mobicash, Wave, Moov | XOF |

### Bank Transfers

| Country | Currency | Settlement Time |
|---------|----------|-----------------|
| Nigeria | NGN | Same day / Next day |
| Kenya | KES | Same day |
| South Africa | ZAR | 1-2 business days |

See [Disbursement Providers](/guides/disbursements/supported-providers) for complete details.

## Disbursement Status

| Status | Description |
|--------|-------------|
| `pending` | Disbursement created, awaiting processing |
| `processing` | Being processed by provider |
| `successful` | Funds delivered to recipient |
| `failed` | Disbursement failed (see error code) |
| `reversed` | Funds returned (recipient issue) |

## Bulk Disbursements

For high-volume disbursements, use our bulk disbursement endpoint:

```bash
curl --location 'https://sandbox-api.payfonte.com/payments/v1/disbursements/bulk' \
  --header 'client-id: YOUR_CLIENT_ID' \
  --header 'client-secret: YOUR_CLIENT_SECRET' \
  --header 'Content-Type: application/json' \
  --data '{
    "batchReference": "BATCH-001",
    "disbursements": [
      {
        "reference": "disbursement-001",
        "amount": 10000,
        "currency": "NGN",
        "provider": "mtn-momo-nigeria",
        "recipient": {
          "name": "John Doe",
          "phoneNumber": "08012345678"
        }
      },
      {
        "reference": "disbursement-002",
        "amount": 15000,
        "currency": "NGN",
        "provider": "mtn-momo-nigeria",
        "recipient": {
          "name": "Jane Smith",
          "phoneNumber": "08087654321"
        }
      }
    ]
  }'
```

## Best Practices

<AccordionGroup>
  <Accordion title="Validate Recipients First" icon="user-check">
    Use name verification endpoints where available to confirm recipient details before sending funds.
  </Accordion>

  <Accordion title="Implement Idempotency" icon="fingerprint">
    Use unique `reference` values and handle duplicate webhook deliveries gracefully.
  </Accordion>

  <Accordion title="Monitor Balance" icon="wallet">
    Ensure sufficient wallet balance before initiating disbursements. Set up low-balance alerts.
  </Accordion>

  <Accordion title="Handle Failures Gracefully" icon="triangle-exclamation">
    Implement retry logic for transient failures. Some failures (invalid account) should not be retried.
  </Accordion>
</AccordionGroup>
