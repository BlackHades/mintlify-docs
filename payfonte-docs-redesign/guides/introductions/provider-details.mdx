# Provider Details - Mobile Money API Specifications

This page provides technical specifications for each **alternative payment method (APM)** and **mobile money API** provider, including decimal place support, authorization types, and amount limits. Use this reference when implementing your **mobile money integration Africa**.

<Note>
  The number of decimal places supported varies by provider. Always format amounts according to the specifications below to avoid transaction failures when using **MTN MoMo API integration**, **M-Pesa API**, **Airtel Money API**, or other **Africa mobile money API** providers.
</Note>

---

## Understanding the Specifications

| Column | Description |
|--------|-------------|
| **Provider** | Mobile Money Operator or payment method name |
| **Slug** | API identifier used in requests |
| **Currency** | ISO currency code for the country |
| **Type** | `payment` (collection) or `disbursement` (disbursement) |
| **Decimals** | Number of decimal places supported in `amount` field |
| **Min Amount** | Minimum transaction amount (in minor units if decimals supported) |
| **Max Amount** | Maximum transaction amount (in minor units if decimals supported) |

### Decimal Places Explained

- **Not supported** = Integers only (e.g., `1000` for 1,000 XOF)
- **2 places** = Two decimal places (e.g., `1000.50` for 1,000.50 GHS)

## Amount Formatting Examples

<CodeGroup>
```javascript XOF (No Decimals)
// For XOF currencies (CFA Franc) - no decimals supported
const amount = 10000; // 10,000 XOF
```

```javascript NGN (2 Decimals)
// For NGN (Nigerian Naira) - 2 decimals supported
const amount = 10000.50; // 10,000.50 NGN
```

```javascript GHS (2 Decimals)
// For GHS (Ghanaian Cedi) - 2 decimals supported
const amount = 500.00; // 500.00 GHS
```
</CodeGroup>

## Dynamic Provider Configuration

You can programmatically fetch available providers and their specifications using the active configuration endpoint:

```bash
curl --location 'https://sandbox-api.payfonte.com/payments/v1/providers' \
  --header 'client-id: YOUR_CLIENT_ID' \
  --header 'client-secret: YOUR_CLIENT_SECRET'
```

This returns real-time information about:
- Available providers for your account
- Supported currencies and decimal places
- Current transaction limits
- Provider status (active/maintenance)
